# Базовый Markdown

> Markdown должен быть настолько легким для чтения и записи, насколько это возможно.
> При этом удобочитаемость ставится превыше всего. 
> Синтаксис Markdown состоит из одних знаков препинания, тщательно отобранных, чтобы разметка была похожа на то что она означает. Например, звездочки вокруг слова в самом деле похожи на выделение. Списки Markdown выглядят как списки. Даже цитаты выглядят как цитируемые отрывки текста, если вы когда-либо использовали электронную почту.

## Абзацы

Пустая строка является разделителем абзацев. Чтобы начать новый абзац, просто дважды нажмите на `Enter`.

## Переносы строк

По умолчанию в Obsidian одно нажатие `Enter` создаёт новую строку в заметке, но при отображении она рассматривается как _продолжение_ того же абзаца.

Чтобы вставить перенос строки _внутри_ абзаца, не начиная новый, вы можете либо добавить **два пробела** в конец строки, либо нажать `Shift+Enter`, чтобы вставить перевод строки.

## Выделение текста: курсив, жирный, выделение маркером, зачеркивание, код

Для выделения текста используем символы подчеркивания или звездочки:

```
одна звездочка или подчеркивание — *курсив*
две звездочки или подчеркивания — **жирный**
жирный курсив — ***три символа***
можно комбинировать: *курсив **полужирный курсив***
```

> одна звездочка или подчеркивание — *курсив*
> две звездочки или подчеркивания — **жирный**
> жирный курсив — ***три символа***
> можно комбинировать: *курсив **полужирный курсив***

Чтобы *зачеркнуть* текст — используем двойные тильды:

```
следующий текст ~~вычеркиваем~~
```

> следующий текст ~~вычеркиваем~~

Аналогично можно выделить текст желтым «маркером» — для этого используем два символа равно:

```
тут ==выделяем== текст
```

> тут ==выделяем== текст

Если требуется использовать моноширинный шрифт (для исходного кода, например), то используем одиночные обратные апострофы (для отдельных слов внутри текста) или отдельные абзацы, ограниченные сверху и снизу строками с тремя обратными апострофами.
