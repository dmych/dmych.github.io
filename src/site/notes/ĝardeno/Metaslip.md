# Metaslip

Хранит карточки с заметками и позволяет связывать их в виде дерева (как в [[ĝardeno/Zettelkasten\|Zettelkasten]]).

С одной стороны, это как раз то, что мне нужно, с другой стороны — слабая функциональность и слабая поддержка Markdown — делают приложение малоупотребительным для серьёзных вещей.

## Дерево карточек

Каждая карточка может иметь родительскую и дочерних (кроме того, можно ссылаться на другие карточки из текста заметки).

Однако, нет (или я не нашёл) способа увидеть дерево карточек — на домашнем экране карточки в порядке их создания (редактирования?)

## Отличия от Zettelkasten

Прежде всего, отсутствует общая последовательность.
В [[ĝardeno/Zettelkasten\|Zettelkasten]] карточки одного уровня упорядочены по времени создания, или по тому, как их создатель их разложил.

В Metaslip карточки все свалены в большую кучу. Их, конечно, можно отсортировать во вкладке Surfing по разным критериям, но это не тот жёсткий порядок, как в Zettelkasten.

## Markdown в Metaslip

Я бы его назвал «ограниченным Markdown» — не вижу, чтобы правильно отображались цитаты и заголовки. По сути, есть только жирный, курсив, неупорядоченный список, математические выражения (LaTeX) и ссылки в двойных квадратных скобках. Не ни картинок, ни блоков кода.
- LaTeX mathematical formulas: `$$ latex formula $$`
- Bold: `**bold**`
- Italic: `*italic*`
- Unordered list: `* unordered list`
- Reference to another card: use `[[card]]` to reference another card.

## Возможности загрузки/выгрузки

Можно выгрузить все карточки в более-менее приличном Markdown.

Передать информацию через меню share в Андроиде нельзя — таким образом можно только восстановиться из бэкапа.

## Синхронизация Metaslip

Реализовано сохранение бэкапов в облачное хранилище через WebDAV.

Можно использовать WebDAV от Яндекс.Диска.

Чтобы синхронизировать изменения на устройство, надо вручную восстановить из бэкапа.

Бэкапы хранятся в бинарном формате.

Это всё не очень удобно, довольно долго, и чревато ошибками, но всё же это рабочая схема.

**Плюсы**
- кроссплатформенность
- минимализм и лаконичность интерфейса
- иерархия заметок и ссылки
- простота использования

Минусы
- слишком бедное подмножество Markdown, отсутствуют вложения
- сложность синхронизации (только вручную, и вероятность ошибки велика)
- глюки клавиатуры (как ни странно, на Маке): странно работают клавиши перемещения в начало/конец строки, нет перемещения по словам, не работает Cmd+Delete для удаления слова...
- собственный формат хранения данных, Markdown только при экспорте
- неясно, насколько приложение поддерживается до сих пор — последнее обновление для Андроида было в июне 2024, для Мака — ещё раньше
- плоская система заголовков — по сути, все заметки находятся в одном namespace (как файлы в папке), поэтому не может быть двух заметок в разных местах иерархии с одинаковыми названиями

## Использование Metaslip
    
- Однако, простота и лаконичность приложения сильно привлекают, несмотря на ограничения и недостатки.
- Приложение спокойно работает на Маке, Андроиде, Айфоне и планшете.
- На компе и на планшете в горизонтальной ориентации приложение отображает заметки в две колонки: слева сама заметка, а справа — все заметки, связанные с ней. Это, конечно, не совсем то отображение, которое мне было бы полезно, но в общем неплохо.
- Конечно, хотелось бы иметь возможность видеть и редактировать параллельно две *разные* заметки, возможно, даже из разных частей иерархии.
